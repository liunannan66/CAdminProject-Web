# 菜单管理：项目立项

前端路由已在 `src/router/elegant/routes.ts` 中注册为 `contract_project_init`。动态权限模式下，需在 **系统管理 → 菜单管理** 中手工添加菜单，并为角色授权。

## 1. 确认「合同管理」目录

在菜单列表中找到 **合同管理** 目录（`routeName` = `contract`，`routePath` = `/contract`），记下其 **Id**（下文记为 `@ContractParentId`）。

若尚无合同目录，先新增：

| 字段 | 值 |
|------|-----|
| 菜单类型 | 目录 (1) |
| 菜单名称 | 合同管理 |
| 路由名称 | contract |
| 路由路径 | /contract |
| 页面组件 | layout.base |
| 图标 | mdi:file-document-edit-outline |
| 排序 | 5 |

## 2. 新增「项目立项」菜单

在 **合同管理** 下新增子菜单：

| 字段 | 值 |
|------|-----|
| 父级菜单 | 合同管理 |
| 菜单类型 | 菜单 (2) |
| 菜单名称 | 项目立项 |
| 路由名称 | contract_project-init |
| 路由路径 | /contract/project-init |
| 页面组件 | view.contract_project-init |
| 图标 | mdi:rocket-launch-outline |
| 排序 | 2 |
| 状态 | 启用 |

保存后，在 **角色管理** 中为管理员角色勾选该菜单。

## 3. 按钮权限（可选，联调后端时使用）

若后端配置了按钮权限，可在菜单下增加按钮项，权限码与前端 `PROJECT_INIT_AUTH` 一致：

- `api:ProjectInit:PageList`
- `api:ProjectInit:Add`
- `api:ProjectInit:Update`
- `api:ProjectInit:Delete`

合同列表「项目立项」按钮使用：`api:Contract:ProjectInit`。

未配置上述按钮时，前端演示模式默认放行。
