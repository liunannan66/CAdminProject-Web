# 产品管理 — 前后端联调说明

## 一、已实现内容

### 后端（MalusAdmin）

| 文件 | 说明 |
|------|------|
| `MalusAdmin.Repository/Model/Business/BdProduct.cs` | 实体 `bd_product` |
| `MalusAdmin.Servers/BusinessManger/Product/ProductService.cs` | 动态 API |
| `MalusAdmin.Servers/BusinessManger/Product/Dto/*` | 分页/入参/启用列表 DTO |

| 接口 | 方法 | 权限码 |
|------|------|--------|
| `/api/Product/PageList` | GET | `api:Product:PageList` |
| `/api/Product/ListEnabled` | GET | `api:Product:ListEnabled` |
| `/api/Product/Add` | POST | `api:Product:Add` |
| `/api/Product/Update` | POST | `api:Product:Update` |
| `/api/Product/Delete/{id}` | GET | `api:Product:Delete` |

### 前端（soybean-admin）

| 文件 | 说明 |
|------|------|
| `src/service/api/basedata.ts` | 产品 CRUD 请求 |
| `src/constants/basedata-product.ts` | 按钮权限码 |
| `src/views/basedata/product/index.vue` | 列表接 API |
| `src/views/basedata/product/modules/product-operate-dialog.vue` | 新增/编辑接 API |

## 二、数据库脚本（按顺序执行）

1. `doc/sql/create-bd_product.sqlserver.sql` — 建表 + 种子数据  
2. `doc/sql/add-product-button-permission.sqlserver.sql` — 修改 `@role_id` 后执行  

执行后请 **重新登录** 以刷新按钮权限缓存。

## 三、本地启动

```bash
# 后端
cd MalusAdmin-master/MalusApi/MalusAdmin.WebApi
dotnet run --launch-profile https

# 前端
cd soybean-admin-element-plus-main
pnpm dev --mode test
```

前端 `.env.test` 中接口地址指向 `https://localhost:7154`。

## 四、验证清单

- [ ] 登录后打开 **基础数据管理 → 产品管理**，列表有数据  
- [ ] 关键字 / 类别 / 状态筛选有效  
- [ ] 新增、编辑、删除成功且刷新后持久化  
- [ ] Swagger：`BusinessManger` 分组下可见 `Product` 接口  

## 五、常见问题

1. **404 Product**  
   确认 `ApiSetup` 已注册 `MalusAdmin.Servers` 程序集，并重启 WebApi。

2. **403 / 无权限**  
   执行权限 SQL，角色分配 `api:Product:*` 后重新登录。

3. **种子数据未导入**  
   `sqlcmd` 中文可能乱码，可在页面手工新增，或用 SSMS 执行建表脚本。

4. **名称+版本重复**  
   接口返回 207：「相同产品名称与版本已存在」。
